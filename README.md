Менеджер паролей GophKeeper

Функции:
1. Хранение бинарных файлов (отправка сжатых chunk на сервер, получение chunk с сервера и преобразование в единый файл); 
2. Синхронизация данных между несколькими авторизованными клиентами одного владельца; 
3. Регистрация, аутентификация и авторизация пользователей (сервер+клиент); 
4. Хранение логина + пароля/банковской карты (хранение приватных данных); 
5. Передача приватных данных владельцу по запросу; 
6. шифрование aes256 для бинарных файлов; 
7. шифрование rsa для любых секретов;

Как запустить:
1.Клонируем проект
2.Переходим в папку проекта
2.Выполняем генерацию сертификатов для сервера и клиента, сборку клиента

Команда - make generate-key;make move-key1;make generate-key;make move-key2;make build-client

3.Выполняем команду docker compose up
4.Убеждаемся что порты 8080 (для сервера) и 5432(для базы данных) сейчас свободны
5.Поднимаем базу данных в docker контейнере

Команда - docker compose up -d go_db

6.Запускаем сервер приложения в docker контейнере

Команда - docker compose up server

7.Переходим в каталог клиента

Команда - cd cmd/client

8.Запускаем клиент

Команда - ./main

9.Регистрируемся
10.Ааутентифицируемся
11.Для отправки файлов на сервер используем папку - files_client_upload в корне, в неё складываем файлы, которые хотим отправить
12.Синхронизация файлов с сервера происходит в эту же папку (files_client_upload)
