# Менеджер паролей GophKeeper

## Функции:
1. Хранение бинарных файлов (отправка сжатых chunk на сервер, получение chunk с сервера и преобразование в единый файл); 
2. Синхронизация данных между несколькими авторизованными клиентами одного владельца; 
3. Регистрация, аутентификация и авторизация пользователей (сервер+клиент); 
4. Хранение логина + пароля/банковской карты (хранение приватных данных); 
5. Передача приватных данных владельцу по запросу; 
6. шифрование aes256 для бинарных файлов; 
7. шифрование rsa для любых секретов;
8. работа по tls протоколу

## Начало работы:
- 1.Клонируем проект;
- 2.Переходим в папку проекта;
- 3.Выполняем генерацию ключей для сервера и клиента, сборку клиента;

```
make generate-keys-rsa
```
- 4.Выполняем генерацию сертификатов сервера и клиента для tls;
```
make generate-cert
```
- 5.Меняем при необходимости secretAuth в internal/server/config/server.json;
- 6.Меняем при необходимости aes256key в internal/client/config/client.json;
- 7.Выполняем команду docker compose up;
- 8.Убеждаемся что порты 8443 (для сервера) и 5432(для базы данных) сейчас свободны;
- 9.Поднимаем базу данных в docker контейнере;

```
docker compose up -d go_db
```

- 10.Запускаем сервер приложения в docker контейнере;

```
docker compose up server
```

- 11.Переходим в каталог клиента;

```
cd cmd/client
```

- 12.Запускаем клиент;

```
./main
```

- 13.Регистрируемся;
- 14.Аутентифицируемся;
- 15.Для отправки файлов на сервер используем папку - files_client_upload в корне.
В неё складываем файлы, которые хотим отправить.
Синхронизация файлов с сервера происходит в эту же папку.
